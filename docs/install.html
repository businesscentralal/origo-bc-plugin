Connect Claude to Business Central

origo-bc wires Claude Desktop up to the Origo Business Central MCP server. You always start from Claude Desktop — what you do next depends on which mode you're in.

Which mode? →  ·  Cowork  ·  Code  ·  Entra app  ·  Versions

Hvaða hamur í Claude Desktop?

Claude Desktop (frá claude.ai/download) býður upp á þrjár leiðir — og origo-bc er ekki sett upp á sama hátt í þeim öllum. Finndu þann sem þú ert í:

Chat Ekki stutt

Venjulegt Claude Desktop spjall. Engin plugin-stuðning — origo-bc virkar ekki hér. Skiptu yfir í Cowork eða Code.

Cowork Mælt með

Skrifstofuverkefni — tala við Claude, leita í BC, undirbúa PEPPOL UBL XML, birtingarhæf svör. Dragðu .plugin-skrá í spjall.

Uppsetning fyrir Cowork →

Code Þróun

Að skrifa kóða — AL, C#, Python, TypeScript, PowerShell eða annað — t.d. integrationir við Cloud Events eða önnur BC-verkefni úr skipanaglugga. Uppsetning með claude plugin marketplace add.

Uppsetning fyrir Code →

Hvað þú færð

  • Sjálfvirk regla (origo-bc-accounting) sem kennir Claude vinnuflæðið hjá Origo — nafnrými, tengiforrit, uppfærslureglur.
  • Þróunarregla (origo-bc-cloud-events) sem hleður Cloud Events API-skilgreiningum, skilaboðagerðum og dæmum gegnum MCP-þjóninn.
  • /origo-bc-setup — fyrsti tengingaraðstoðarmaður.
  • /origo-bc-add-env — bætir við fleiri BC-umhverfum.
  • /origo-bc-list-environments — birtir öll tengd umhverfi.
  • /origo-bc-switch-company — skipta um sjálfgefið fyrirtæki.

Áður en þú byrjar

Node.js 18 eða nýrri

Bæði Cowork- og Code-leiðin keyra tengingu í gegnum lítið Node-ferli. Staðfestu að Node sé uppsett: 

node --version

Ef skipunin finnst ekki — sæktu LTS-útgáfuna frá nodejs.org og endurræstu Claude Desktop eftir uppsetningu.

Ef þú ert í Cowork

Einfaldasta leiðin er að sækja tilbúna .plugin-skrá og draga hana inn í spjallið.

1. Sækja skrána

Smelltu á hnappinn hér að neðan í vafra (Edge, Chrome, Firefox — ekki inni í Cowork). Vafrinn vistar origo-bc-cowork.plugin í Downloads-möppuna. Skráin er 10–20 KB.

Sækja origo-bc-cowork.plugin Eða veldu fasta útgáfu →

Ekki líma slóðina inn í Cowork sem skilaboð URL-ið er aðeins niðurhalsslóð. Cowork getur ekki sett upp plugin af URL-i — þú þarft að hafa skrána sjálfa á tölvunni þinni áður en þú heldur áfram.

2. Setja upp í Cowork

Tvær leiðir — báðar virka, veldu þá sem hentar:

Leið A — draga skrána í spjall

  1. Opnaðu File Explorer og farðu í Downloads.
  2. Opnaðu nýtt spjall í Cowork við hliðina á.
  3. Dragðu skrána origo-bc-cowork.plugin beint úr File Explorer og slepptu henni í innsláttarreitinn (þar sem þú skrifar skilaboð).
  4. Ýttu á Send. Ekki hætta við — þú þarft að senda skilaboðin.
  5. Claude svarar með uppsetningarkorti — smelltu á Install.

Leið B — gegnum stillingar

  1. Í Cowork: Settings → Plugins.
  2. Smelltu á Install from file.
  3. Flettu í Downloads og veldu origo-bc-cowork.plugin.
  4. Staðfestu uppsetninguna.
Ef uppsetningarkortið birtist ekki Athugaðu þrennt: (1) að þú sért í Cowork-ham, ekki venjulegu Claude Desktop spjalli — titillinn á glugganum ætti að segja „Cowork"; (2) að þú hafir dregið inn skrána, ekki URL-ið; (3) að þú hafir ýtt á Send, ekki bara hætt við.

Ef þú ert í Code

Claude Code (CLI-hamurinn) notar ekki .plugin-skrá. Fyrst þarftu að setja upp claude CLI-tólið sjálft, og keyra síðan tvær skipanir til að bæta við origo-bc.

1. Setja upp Claude Code CLI

Ef þú ert ekki með claude skipunina á PATH, settu hana upp fyrst:

macOS

# Leið A — Homebrew (mælt með)
brew install claude

# Leið B — npm
npm install -g @anthropic-ai/claude-code

Windows

# npm (krefst Node.js 18+)
npm install -g @anthropic-ai/claude-code

Staðfestu uppsetningu: 

claude --version

Ef skipunin finnst ekki, lokaðu og opnaðu skipanagluggann aftur (eða endurræstu VS Code) svo PATH-breytan uppfærist.

2. Bæta við origo-bc plugin

Keyrðu tvær skipanir — Code hleður origo-markaðstorginu beint af opinberu GitHub-repo-inu okkar: 

claude plugin marketplace add businesscentralal/origo-bc-plugin
claude plugin install origo-bc@origo

Fyrri skipunin skráir origo-markaðstorgið. Seinni skipunin sækir nýjustu útgáfu af origo-bc og setur hana upp í ~/.claude/plugins/. Endurræstu Claude Code (eða opnaðu nýja sessjón) til að skilgreiningin hlaðist.

Virkar líka í Cowork Ef þú setur upp plugin-ið í Claude Code verður það einnig virkt í Claude Cowork — þú þarft ekki að setja það upp sérstaklega í báðum hömum.

3. Uppfæra plugin

Þegar ný útgáfa kemur út, keyrðu: 

claude plugin update origo-bc@origo

Endurræstu Claude Code (eða opnaðu nýja sessjón) eftir uppfærslu.

Til að fjarlægja: claude plugin uninstall origo-bc

Forkrafa — Entra App Registration

MCP-þjónninn á dynamics.is/api/mcp auðkennir sig með Microsoft Entra-skráningu (áður Azure AD). Þú þarft þrjú gildi áður en þú keyrir /origo-bc-setup: Tenant ID, Client ID og Client Secret. Skráninguna þarf líka að tengja inn í BC sem Microsoft Entra Application með réttum heimildarsamstæðum.

1. Azure-hliðin — búa til app-skráningu

Við fylgjum staðalmynstri Microsoft fyrir „Business Central Web Service Client" — sami grunnur og Microsoft notar í eigin dæmum. Uppsetningin styður bæði innskráningarflæði (delegated, fyrir notanda við lyklaborðið) og ósjálfvirkt flæði (app-only / client-credentials, fyrir MCP-þjóninn).

1a. Nýskrá app

  1. Farðu inn í portal.azure.com → Microsoft Entra ID → App registrations → New registration.
  2. Name: Business Central Web Service Client — eða <fyrirtæki> BC — Claude ef þú vilt merkja við sérstakt umhverfi.
  3. Supported account types: bæði virka — Accounts in this organizational directory only (Single tenant) er einfaldast og duglegt þegar appið þjónar aðeins eigið fyrirtæki. Accounts in any organizational directory (Multitenant) er gagnlegt ef sama skráning á að gilda yfir mörg umhverfi eða hjá þjónustuaðila. Dæmið sem við vísum í notar AzureADMultipleOrgs, en single-tenant virkar alveg eins fyrir flesta.
  4. Redirect URI: veldu Public client/native (mobile & desktop) og sláðu inn: 
    https://businesscentral.dynamics.com/OAuthLanding.htm
  5. Smelltu á Register.

1b. Authentication — leyfa public client flows

  1. Opnaðu Authentication-flipann á nýja app-inu.
  2. Neðst, undir Advanced settings: Allow public client flows → Yes.
  3. Smelltu á Save.

1c. API permissions — fjórar heimildir

Í API permissions-flipanum smellirðu á Add a permission og bætir við öllum fjórum hér að neðan. Þrjár eru Delegated (í nafni notanda), ein er Application (app-only — skilyrði fyrir MCP-þjóninn).

Microsoft Graph

  • User.Read — Delegated.

Dynamics 365 Business Central

  • Financials.ReadWrite.All — Delegated (les/skrifa BC-gögn í nafni notanda).
  • user_impersonation — Delegated (stöðluð BC-beinisaðganga).
  • app_access — Application (ósjálfvirkur aðgangur, MCP-þjónninn).
  1. Þegar allar fjórar eru komnar í listann, smelltu á Grant admin consent for <leigjandi> og staðfestu. Status-dálkurinn á að sýna græna gátmerkingu fyrir allar heimildirnar.

1d. Certificates & secrets — nýtt client secret

  1. Certificates & secrets → Client secrets → New client secret.
  2. Description: notaðu nafnavenju sem segir hver bjó lykilinn til og á hvaða tíma (t.d. ORI1058 fylgir Origo-innri mynstri). Hjálpar þegar þarf að skipta út lykli seinna.
  3. Expires: mælt með 12–24 mánuðum (Azure leyfir ekki meira en 24).
  4. Smelltu á Add. Afritaðu gildið strax — það birtist ekki aftur, aðeins hint (t.d. Z26…) og Secret ID.

1e. Afrita það sem þú þarft

Fyrir /origo-bc-setup þarftu þrjú gildi — afritaðu þau úr Azure:

  • Tenant ID — Overview → Directory (tenant) ID.
  • Client ID — Overview → Application (client) ID.
  • Client Secret — gildið úr skrefi 1d (ekki Secret ID).
Ekki deila client-secret-inu Geymdu gildið á öruggum stað (password manager). Ef það týnist skaltu búa til nýtt — það er ekki hægt að birta gamalt aftur.
Lágmarksútgáfan (aðeins MCP) Ef þú notar appið einungis fyrir Claude MCP-tengingu (aldrei fyrir innskráðan notanda), dugar einn permission: app_access (Application). Redirect URI og Allow public client flows eru þá heldur ekki nauðsynleg — en það er samt þægilegt að hafa þau með strax svo sama skráning virki líka fyrir interactive tól (t.d. BC-appið sjálft) síðar.

2. BC-hliðin — skrá appið og gefa heimildir

  1. Opnaðu Business Central sem kennitölunotandi með kerfisstjóraheimildir.
  2. Leitaðu (Alt+Q) að Microsoft Entra Applications og opnaðu síðuna.
  3. New → nýtt spjaldið lítur út eins og skjámyndin hér að neðan:

General

  • Client ID: líma Application (client) ID úr Azure.
  • Description: stutt heiti (t.d. Cloud Events Tests).
  • State: Enabled.
  1. Í flipanum User Permission Sets bætir þú við a.m.k. eftirfarandi samstæðum:

Lágmarksheimildir

  • CLOUD EVENTS API — Cloud Events Base / System. Skylda. Þetta er samstæðan sem gefur Claude aðgang að Cloud Events endapunktunum og MCP-tólinu sjálfu.
  • D365 BASIC — Base Application / System. Grunnaðgangur að BC (lestur á yfirlitum, tenglum, númeraröðum).

Gagnaheimildir (veldu eftir notkun)

  • SUPER (DATA) — System Application / System. Fullur aðgangur að BC-gögnum. Einfaldast að setja upp, en mjög gjafmilt — ekki nota fyrir utanaðkomandi aðila.
  • Eða sérsniðnar samstæður: t.d. D365 READ + D365 SALES ef Claude á bara að lesa og skoða sölur; D365 FIN fyrir fjármálavinnslur; o.s.frv.

Heimildirnar sem þú bætir við hér stjórna hvaða gögnum Claude getur séð og breytt í þessu tiltekna fyrirtæki. Að breyta þeim seinna krefst þess að State sé sett á Disabled fyrst.

3. Veita samþykki (Grant Consent)

  1. Á Entra Application-spjaldinu smellirðu á Grant Consent (efst).
  2. BC beinir þér yfir á Microsoft-innskráningu — skráðu þig inn sem kerfisstjóri leigjandans (tenant admin) og samþykktu.
  3. Ef samþykki tekst snýrðu aftur í BC með „Consent granted"-skilaboðum. Appið er núna tilbúið.
Prófunar- vs. framleiðsluumhverfi Búðu til aðskilda Entra-skráningu fyrir hvert umhverfi (t.d. cronus, prod) og aðskildar Entra Application-færslur í BC. Þú getur síðan tengt þau í Claude í gegnum /origo-bc-add-env og skipt á milli án þess að blanda saman leyfum.

Which Claude Desktop mode?

Claude Desktop (from claude.ai/download) ships three modes — and origo-bc is installed differently in each one. Find the one you're in:

Chat Not supported

The regular Claude Desktop chat. No plugin support — origo-bc won't run here. Switch to Cowork or Code.

Cowork Recommended

Office work — talking to Claude, searching BC, preparing PEPPOL UBL XML, presentable replies. Install by dropping a .plugin file into chat.

Install for Cowork →

Code Development

Writing code — AL, C#, Python, TypeScript, PowerShell, or anything else — for Cloud Events integrations or other BC-related work from the terminal. Install with claude plugin marketplace add.

Install for Code →

What you get

  • An always-on rule (origo-bc-accounting) that teaches Claude the Origo BC workflow — namespaces, connection formats, update rules.
  • A development rule (origo-bc-cloud-events) that loads the Cloud Events API definitions, message types, and examples via the MCP server.
  • /origo-bc-setup — first-time connection wizard.
  • /origo-bc-add-env — add additional BC tenants.
  • /origo-bc-list-environments — list everything connected.
  • /origo-bc-switch-company — change the default company.

Before you start

Node.js 18 or newer

Both Cowork and Code drive the connection through a small Node process. Check your install: 

node --version

If the command isn't found, grab the LTS build from nodejs.org and restart Claude Desktop afterwards.

If you're in Cowork

The simplest path is to download the prebuilt .plugin file and drop it into chat.

1. Download the file

Click the button below in a web browser (Edge, Chrome, Firefox — not inside Cowork). The browser saves origo-bc-cowork.plugin into your Downloads folder. The file is 10–20 KB.

Download origo-bc-cowork.plugin Or pick a pinned version →

Don't paste the URL into Cowork as a message The URL is just a download link. Cowork can't install plugins from URLs — you need the file itself on your machine before continuing.

2. Install in Cowork

Two ways — either works, pick whichever is handier:

Option A — drop the file into chat

  1. Open File Explorer / Finder and go to Downloads.
  2. Open a new Cowork chat alongside it.
  3. Drag the origo-bc-cowork.plugin file directly from File Explorer into the chat message input (where you type).
  4. Hit Send. Don't cancel — you need to actually send the message.
  5. Claude replies with an install card — click Install.

Option B — via Settings

  1. In Cowork: Settings → Plugins.
  2. Click Install from file.
  3. Browse to Downloads and pick origo-bc-cowork.plugin.
  4. Confirm the install.
If no install card appears Check three things: (1) you're in Cowork mode, not the regular Claude Desktop chat — the window title should say "Cowork"; (2) you dropped the file, not the URL; (3) you hit Send instead of cancelling.

If you're in Code

Claude Code (the CLI mode) doesn't use the .plugin file. First install the claude CLI itself, then run two commands to add origo-bc.

1. Install the Claude Code CLI

If you don't have the claude command on your PATH yet, install it first:

macOS

# Option A — Homebrew (recommended)
brew install claude

# Option B — npm
npm install -g @anthropic-ai/claude-code

Windows

# npm (requires Node.js 18+)
npm install -g @anthropic-ai/claude-code

Verify the install: 

claude --version

If the command isn't found, close and reopen your terminal (or restart VS Code) so the updated PATH takes effect.

2. Add the origo-bc plugin

Run two terminal commands — Code pulls the origo marketplace straight from our public GitHub repo: 

claude plugin marketplace add businesscentralal/origo-bc-plugin
claude plugin install origo-bc@origo

The first command registers the origo marketplace. The second pulls the latest origo-bc build and installs it into ~/.claude/plugins/. Restart Claude Code (or open a new session) so the definitions load.

Also active in Cowork If you install the plugin in Claude Code, it will also be active in Claude Cowork — you don't need to install it separately in both modes.

3. Update the plugin

When a new version is released, run: 

claude plugin update origo-bc@origo

Restart Claude Code (or open a new session) after updating.

To remove: claude plugin uninstall origo-bc

Prerequisite — Entra App Registration

The MCP server at dynamics.is/api/mcp authenticates against Microsoft Entra (formerly Azure AD). Before you run /origo-bc-setup you need three values: Tenant ID, Client ID, and Client Secret. You also need to register the app inside BC as a Microsoft Entra Application with the right permission sets.

1. Azure side — create the app registration

We follow Microsoft's standard template for a "Business Central Web Service Client" — the same baseline Microsoft uses in their own samples. The registration supports both delegated (a signed-in user) and app-only / client-credentials (the MCP server running unattended).

1a. Register a new app

  1. Open portal.azure.com → Microsoft Entra ID → App registrations → New registration.
  2. Name: Business Central Web Service Client — or <company> BC — Claude if you want to tag a specific environment.
  3. Supported account types: either works — Accounts in this organizational directory only (Single tenant) is the simplest choice when the app only serves your own organization. Accounts in any organizational directory (Multitenant) is useful if the same registration should serve multiple environments or a service provider's customers. The reference example uses AzureADMultipleOrgs, but single-tenant is fine for most installations.
  4. Redirect URI: pick Public client/native (mobile & desktop) and enter: 
    https://businesscentral.dynamics.com/OAuthLanding.htm
  5. Click Register.

1b. Authentication — allow public client flows

  1. On the new app, open the Authentication tab.
  2. At the bottom, under Advanced settings: Allow public client flows → Yes.
  3. Click Save.

1c. API permissions — four permissions

In the API permissions tab click Add a permission and add all four below. Three are Delegated (on behalf of the signed-in user), one is Application (app-only — required for the MCP server).

Microsoft Graph

  • User.Read — Delegated.

Dynamics 365 Business Central

  • Financials.ReadWrite.All — Delegated (read/write BC data on the user's behalf).
  • user_impersonation — Delegated (the stock BC access-as-user scope).
  • app_access — Application (unattended access, used by the MCP server).
  1. Once all four show in the list, click Grant admin consent for <tenant> and confirm. The Status column should turn to a green checkmark for every permission.

1d. Certificates & secrets — create a client secret

  1. Certificates & secrets → Client secrets → New client secret.
  2. Description: use a naming convention so you know who created the secret and when (e.g. ORI1058 follows Origo's internal pattern). Makes it easier to rotate later.
  3. Expires: 12–24 months recommended (Azure caps at 24).
  4. Click Add. Copy the value immediately — only a hint (e.g. Z26…) and the Secret ID remain visible afterwards.

1e. Copy the values you need

For /origo-bc-setup you need three values. Grab them from Azure:

  • Tenant ID — Overview → Directory (tenant) ID.
  • Client ID — Overview → Application (client) ID.
  • Client Secret — the value from step 1d (not the Secret ID).
Don't share the client secret Keep the value in a password manager. If it's lost, generate a new one — old values can't be displayed again.
Minimal version (MCP only) If you only use this app for the Claude MCP connection (never for a signed-in user), a single permission is enough: app_access (Application). The redirect URI and Allow public client flows aren't strictly needed in that case either — but it's handy to add them up front so the same registration can be reused by interactive tools (the BC app itself, etc.) later.

2. BC side — register the app and grant permissions

  1. Open Business Central signed in as an admin user.
  2. Search (Alt+Q) for Microsoft Entra Applications and open the page.
  3. New → the new card looks like this:

General

  • Client ID: paste the Application (client) ID from Azure.
  • Description: a short name (e.g. Cloud Events Tests).
  • State: Enabled.
  1. On the User Permission Sets tab add, at minimum, the following:

Minimum permissions

  • CLOUD EVENTS API — Cloud Events Base / System. Required. This is the permission set that grants Claude access to the Cloud Events endpoints and the MCP tooling itself.
  • D365 BASIC — Base Application / System. Baseline BC access (reading lists, links, number series).

Data permissions (pick based on use)

  • SUPER (DATA) — System Application / System. Full access to BC data. Easiest setup, but very generous — don't use for third parties.
  • Or scoped sets: e.g. D365 READ + D365 SALES for read-only sales review; D365 FIN for finance workflows; etc.

The permissions you attach here control what Claude can see and change in this particular company. Changing them later requires setting State back to Disabled first.

3. Grant consent

  1. On the Entra Application card, click Grant Consent (top bar).
  2. BC redirects you to Microsoft — sign in as a tenant admin and approve.
  3. On success you're returned to BC with a "Consent granted" message. The app is now ready.
Separate registrations per environment Create a distinct Entra app (and BC Entra Application record) for each environment — e.g. cronus, prod. You can then register each one in Claude via /origo-bc-add-env and switch between them without mixing permissions.

First connection (both Cowork and Code)

Start a fresh chat / session and type:

/origo-bc-setup

The wizard asks for:

  1. MCP URL — https://dynamics.is/api/mcp by default.
  2. Tenant ID — Directory (tenant) ID from Azure.
  3. Client ID — Application (client) ID from Azure.
  4. Client Secret — the value you copied in step 1.
  5. Environment — Production, Sandbox, or a named sandbox (e.g. cronus).
  6. Company name — as it appears in BC.

Claude then builds the correct connection blob (plain:<base64> on most systems, DPAPI-wrapped on Windows) and runs setup_origo_bc_environment. The MCP server appears in Claude's tool list.

Adding more environments

If you work across multiple BC companies:

/origo-bc-add-env

Each connection gets its own handle (e.g. prod, test, cronus) and you can switch between them without re-running setup.

Change the default company

/origo-bc-switch-company

List everything connected

/origo-bc-list-environments

Troubleshooting

node: command not found

Node.js isn't on PATH. Install LTS from nodejs.org and restart Cowork.

MCP tools don't show up

Quit Cowork fully (tray icon → Quit) and relaunch. MCP servers are only loaded at startup.

Wrong company in answers

Run /origo-bc-switch-company and pick the right one. The change only affects the current chat unless you rerun /origo-bc-setup to make it stick.

"Unauthorized" from the MCP server

Credentials expired or were wrong. Run /origo-bc-setup again and paste fresh credentials.
