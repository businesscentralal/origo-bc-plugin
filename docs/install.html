Connect Claude to Business Central

origo-bc wires Claude Desktop up to the Origo Business Central MCP server. You always start from Claude Desktop — what you do next depends on which mode you're in.

Which mode? →  ·  Cowork  ·  Code  ·  Versions

Hvaða hamur í Claude Desktop?

Claude Desktop (frá claude.ai/download) býður upp á þrjár leiðir — og origo-bc er ekki sett upp á sama hátt í þeim öllum. Finndu þann sem þú ert í:

Chat Ekki stutt

Venjulegt Claude Desktop spjall. Engin plugin-stuðning — origo-bc virkar ekki hér. Skiptu yfir í Cowork eða Code.

Cowork Mælt með

Skrifstofuverkefni — tala við Claude, leita í BC, undirbúa PEPPOL UBL XML, birtingarhæf svör. Dragðu .plugin-skrá í spjall.

Uppsetning fyrir Cowork →

Code Þróun

Að skrifa AL-kóða, búa til Cloud Events integrationir, keyra CLI-skipanir. Uppsetning með claude plugin marketplace add.

Uppsetning fyrir Code →

Hvað þú færð

  • Sjálfvirk regla (bc-mcp-connection-rules) sem kennir Claude vinnuflæðið hjá Origo — nafnrými, tengiforrit, uppfærslureglur.
  • /origo-bc-setup — fyrsti tengingaraðstoðarmaður.
  • /origo-bc-add-env — bætir við fleiri BC-umhverfum.
  • /origo-bc-list-environments — birtir öll tengd umhverfi.
  • /origo-bc-switch-company — skipta um sjálfgefið fyrirtæki.

Áður en þú byrjar

Node.js 18 eða nýrri

Bæði Cowork- og Code-leiðin keyra tengingu í gegnum lítið Node-ferli. Staðfestu að Node sé uppsett: 

node --version

Ef skipunin finnst ekki — sæktu LTS-útgáfuna frá nodejs.org og endurræstu Claude Desktop eftir uppsetningu.

Ef þú ert í Cowork

Einfaldasta leiðin er að sækja tilbúna .plugin-skrá og draga hana inn í spjallið.

1. Sækja skrána

Smelltu á hnappinn hér að neðan í vafra (Edge, Chrome, Firefox — ekki inni í Cowork). Vafrinn vistar origo-bc.plugin í Downloads-möppuna. Skráin er 20–40 KB.

Sækja origo-bc.plugin Eða veldu fasta útgáfu →

Ekki líma slóðina inn í Cowork sem skilaboð URL-ið er aðeins niðurhalsslóð. Cowork getur ekki sett upp plugin af URL-i — þú þarft að hafa skrána sjálfa á tölvunni þinni áður en þú heldur áfram.

2. Setja upp í Cowork

Tvær leiðir — báðar virka, veldu þá sem hentar:

Leið A — draga skrána í spjall

  1. Opnaðu File Explorer og farðu í Downloads.
  2. Opnaðu nýtt spjall í Cowork við hliðina á.
  3. Dragðu skrána origo-bc.plugin beint úr File Explorer og slepptu henni í innsláttarreitinn (þar sem þú skrifar skilaboð).
  4. Ýttu á Send. Ekki hætta við — þú þarft að senda skilaboðin.
  5. Claude svarar með uppsetningarkorti — smelltu á Install.

Leið B — gegnum stillingar

  1. Í Cowork: Settings → Plugins.
  2. Smelltu á Install from file.
  3. Flettu í Downloads og veldu origo-bc.plugin.
  4. Staðfestu uppsetninguna.
Ef uppsetningarkortið birtist ekki Athugaðu þrennt: (1) að þú sért í Cowork-ham, ekki venjulegu Claude Desktop spjalli — titillinn á glugganum ætti að segja „Cowork"; (2) að þú hafir dregið inn skrána, ekki URL-ið; (3) að þú hafir ýtt á Send, ekki bara hætt við.

Ef þú ert í Code

Claude Code (CLI-hamurinn) notar ekki .plugin-skrá. Keyrðu þess í stað tvær skipanir í skipanaglugga — Code hleður origo-markaðstorginu beint af Azure-blobbinum: 

claude plugin marketplace add https://origopublic.blob.core.windows.net/resources/mcp
claude plugin install origo-bc@origo

Fyrri skipunin skráir origo-markaðstorgið. Seinni skipunin sækir nýjustu útgáfu af origo-bc og setur hana upp í ~/.claude/plugins/. Endurræstu Claude Code til að skilgreiningin hlaðist.

Til að uppfæra seinna: claude plugin update origo-bc@origo. Til að fjarlægja: claude plugin uninstall origo-bc.

Bætir við Cloud Events fyrir AL-þróun

Ef þú ert að skrifa AL-kóða sem notar Cloud Events viðbótina hjá Origo, þá þarftu líka Cloud Events-þekkingarregluna. Hún lýsir atburðakerfinu (hook-point-um, payload-sniði, uppsetningarreglum) og leiðir Claude gegnum rétt mynstur.

Reglan liggur á:

https://origopublic.blob.core.windows.net/help/Cloud Events/bc27/en-US/SKILL.md

Þegar þú ert tengd/ur við BC í gegnum /origo-bc-setup, hleður Claude reglunni sjálfkrafa gegnum bc-cloud-events remote-loader-skilgreininguna í MCP-Skills. Ef þú vilt hafa regluna staðbundna (fyrir offline-vinnu), vistaðu skrána sem ~/.claude/skills/bc-cloud-events/SKILL.md.

Fyrsta tenging (bæði Cowork og Code)

Byrjaðu nýtt spjall / sessjón og skrifaðu:

/origo-bc-setup

Aðstoðarmaðurinn spyr þig um:

  1. MCP-slóð — https://dynamics.is/api/mcp er sjálfgefið.
  2. Fyrirtækisnafn — eins og það birtist í BC (t.d. Origo hf.).
  3. Sjálfgefna auðkenninguna — Claude setur upp rétt format fyrir þig.

Þegar stofnunin er tilbúin sérðu setup_origo_bc_environment keyrt og MCP-þjónninn birtist í tólalista Claude.

Bæta við fleiri umhverfum

Ef þú vinnur með fleira en eitt BC-fyrirtæki:

/origo-bc-add-env

Hver tenging fær sitt nafn (t.d. prod, test, cronus) og þú getur skipt á milli þeirra án þess að endurstilla.

Skipta um sjálfgefið fyrirtæki

/origo-bc-switch-company

Sjá öll tengd umhverfi

/origo-bc-list-environments

Bilanagreining

node: command not found

Node.js er ekki á PATH. Settu upp LTS frá nodejs.org og endurræstu Cowork.

MCP-tólin birtast ekki

Lokaðu Cowork alveg (táknið í kerfisbakka → Quit) og opnaðu aftur. MCP-þjónar sækjast aðeins við ræsingu.

Rangt fyrirtæki í svörum

Keyrðu /origo-bc-switch-company og veldu rétt fyrirtæki. Þetta hefur aðeins áhrif á virka spjallið ef þú sendir kommandoið þar — til að gera breytinguna varanlega, sjá /origo-bc-setup.

„Unauthorized" frá MCP-þjóninum

Auðkenning er útrunnin eða röng. Keyrðu /origo-bc-setup aftur og sláðu inn nýja auðkenningu.

Útgáfur

Nýjasta slóðin hér að ofan er alltaf endurskrifuð við hverja útgáfu. Ef þú vilt festa þig við tiltekna útgáfu (t.d. fyrir rekjanleika), notaðu slóð á borð við:

https://origopublic.blob.core.windows.net/resources/mcp/origo-bc-0.1.1.plugin

Útgáfunúmerið má finna í plugin.json innan origo-bc.plugin-skrárinnar, og nýjasta útgáfan er alltaf sú sem er hlaðin upp á slóðina efst á síðunni.

Aðstoð

Hafðu samband við service@origo.is ef þú finnur villu eða þarft hjálp við tengingu. Viðbótin er opinn hugbúnaður frá Origo hf. og allir notendur Business Central mega nota hana.

Which Claude Desktop mode?

Claude Desktop (from claude.ai/download) ships three modes — and origo-bc is installed differently in each one. Find the one you're in:

Chat Not supported

The regular Claude Desktop chat. No plugin support — origo-bc won't run here. Switch to Cowork or Code.

Cowork Recommended

Office work — talking to Claude, searching BC, preparing PEPPOL UBL XML, presentable replies. Install by dropping a .plugin file into chat.

Install for Cowork →

Code Development

Writing AL code, building Cloud Events integrations, running CLI commands. Install with claude plugin marketplace add.

Install for Code →

What you get

  • An always-on rule (bc-mcp-connection-rules) that teaches Claude the Origo BC workflow — namespaces, connection formats, update rules.
  • /origo-bc-setup — first-time connection wizard.
  • /origo-bc-add-env — add additional BC tenants.
  • /origo-bc-list-environments — list everything connected.
  • /origo-bc-switch-company — change the default company.

Before you start

Node.js 18 or newer

Both Cowork and Code drive the connection through a small Node process. Check your install: 

node --version

If the command isn't found, grab the LTS build from nodejs.org and restart Claude Desktop afterwards.

If you're in Cowork

The simplest path is to download the prebuilt .plugin file and drop it into chat.

1. Download the file

Click the button below in a web browser (Edge, Chrome, Firefox — not inside Cowork). The browser saves origo-bc.plugin into your Downloads folder. The file is 20–40 KB.

Download origo-bc.plugin Or pick a pinned version →

Don't paste the URL into Cowork as a message The URL is just a download link. Cowork can't install plugins from URLs — you need the file itself on your machine before continuing.

2. Install in Cowork

Two ways — either works, pick whichever is handier:

Option A — drop the file into chat

  1. Open File Explorer / Finder and go to Downloads.
  2. Open a new Cowork chat alongside it.
  3. Drag the origo-bc.plugin file directly from File Explorer into the chat message input (where you type).
  4. Hit Send. Don't cancel — you need to actually send the message.
  5. Claude replies with an install card — click Install.

Option B — via Settings

  1. In Cowork: Settings → Plugins.
  2. Click Install from file.
  3. Browse to Downloads and pick origo-bc.plugin.
  4. Confirm the install.
If no install card appears Check three things: (1) you're in Cowork mode, not the regular Claude Desktop chat — the window title should say "Cowork"; (2) you dropped the file, not the URL; (3) you hit Send instead of cancelling.

If you're in Code

Claude Code (the CLI mode) doesn't use the .plugin file. Run two terminal commands instead — Code pulls the origo marketplace straight from the Azure blob: 

claude plugin marketplace add https://origopublic.blob.core.windows.net/resources/mcp
claude plugin install origo-bc@origo

The first command registers the origo marketplace. The second pulls the latest origo-bc build and installs it into ~/.claude/plugins/. Restart Claude Code so the definitions load.

To update later: claude plugin update origo-bc@origo. To remove: claude plugin uninstall origo-bc.

Add Cloud Events for AL development

If you're writing AL code that uses Origo's Cloud Events extension, you'll also want the Cloud Events knowledge skill. It documents the event system (hook points, payload shapes, setup rules) and guides Claude through the right patterns.

The skill lives at:

https://origopublic.blob.core.windows.net/help/Cloud Events/bc27/en-US/SKILL.md

Once you're connected to BC via /origo-bc-setup, Claude loads the skill automatically through the bc-cloud-events remote-loader entry in MCP-Skills. If you prefer a local copy (e.g. for offline work), save the file as ~/.claude/skills/bc-cloud-events/SKILL.md.

First connection (both Cowork and Code)

Start a fresh chat / session and type:

/origo-bc-setup

The wizard asks for:

  1. MCP URL — https://dynamics.is/api/mcp by default.
  2. Company name — as it appears in BC (e.g. Origo hf.).
  3. Credentials — Claude builds the correct connection blob for you.

Once the environment is set up, you'll see setup_origo_bc_environment run and the MCP server appear in Claude's tool list.

Adding more environments

If you work across multiple BC companies:

/origo-bc-add-env

Each connection gets its own handle (e.g. prod, test, cronus) and you can switch between them without re-running setup.

Change the default company

/origo-bc-switch-company

List everything connected

/origo-bc-list-environments

Troubleshooting

node: command not found

Node.js isn't on PATH. Install LTS from nodejs.org and restart Cowork.

MCP tools don't show up

Quit Cowork fully (tray icon → Quit) and relaunch. MCP servers are only loaded at startup.

Wrong company in answers

Run /origo-bc-switch-company and pick the right one. The change only affects the current chat unless you rerun /origo-bc-setup to make it stick.

"Unauthorized" from the MCP server

Credentials expired or were wrong. Run /origo-bc-setup again and paste fresh credentials.

Versions

The latest URL above is overwritten on every release. If you need to pin a specific version (for reproducibility), use a URL like:

https://origopublic.blob.core.windows.net/resources/mcp/origo-bc-0.1.1.plugin

Version numbers live in plugin.json inside the origo-bc.plugin file — whatever is linked at the top of this page is always the latest release.
